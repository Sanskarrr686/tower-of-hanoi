Why space complexity differs between the two approaches:
Recursive — O(n): Every call to hanoi(k) is suspended while its two sub-calls run, so the OS call stack accumulates exactly n live frames simultaneously at maximum depth — one per level of recursion. This unavoidable overhead grows linearly with disk count, giving O(n) auxiliary space.
Iterative — O(1): The true iterative solution exploits a mathematical pattern: on step t, move disk v₂(t) (the 2-adic valuation of the step number) between pegs determined by parity. Only a handful of integer variables are needed regardless of n, so auxiliary space stays at O(1) — flat and constant.
Key insight: The recursive call stack is the hidden cost — eliminating it entirely (not just making it explicit) is what achieves true O(1) space in the iterative approach.

Recursive method

Divides the problem: move n−1 disks away, move the largest, then move n−1 back. Uses the call stack to track state. Each call adds one frame — max depth = n.

void hanoi(int n, src, dst, aux) {
  if (n == 0) return; // base case
  hanoi(n-1, src, aux, dst); // phase 1
  print("disk n: src→dst");    // phase 2
  hanoi(n-1, aux, dst, src); // phase 3
}

Iterative method

Uses a manual stack (array) to simulate the call stack. Pushes sub-problems as frames and processes them without any actual recursion. No function call overhead.

void hanoiIter(int n, src, dst, aux) {
  stack = []; // explicit stack
  stack.push({n, src, dst, aux});
  while (stack.length > 0) {
    frame = stack.pop();
    if (frame.n == 0) continue;
    // push 3 operations in reverse
    stack.push({n-1, aux, dst, src});
    stack.push({0, src, dst, 'MOVE'});
    stack.push({n-1, src, aux, dst});
  }
}
